# Data Model: Task CRUD POC

## Entity: Task

Represents one item in the task list.

### Fields

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | Long | Yes | Generated by the backend; immutable after creation |
| `title` | String | Yes | Must contain at least one non-whitespace character |
| `description` | String | No | May be absent, null, empty, or whitespace |
| `status` | TaskStatus | Yes | Defaults to `PENDING`; allowed values: `PENDING`, `COMPLETED` |
| `createdAt` | Instant | Yes | Generated at creation time; immutable after creation |

### Validation

- `title` is required for create and update.
- `title` must be trimmed for validation; whitespace-only titles are invalid.
- `description` is optional and does not affect validity.
- `status` is not accepted from create requests; new tasks start as `PENDING`.
- `createdAt` is generated by the backend and is not accepted from requests.

### State Transitions

```text
PENDING -> COMPLETED
COMPLETED -> COMPLETED
```

The POC only requires completing a task. Reopening a completed task is outside the current scope.

### API Representations

#### TaskRequest

Used for create and full update of editable fields.

| Field | Required | Notes |
|-------|----------|-------|
| `title` | Yes | Non-blank |
| `description` | No | Optional |

#### TaskResponse

Returned by list, create, detail, update, complete, and delete-independent reads.

| Field | Notes |
|-------|-------|
| `id` | Task identifier |
| `title` | Current title |
| `description` | Current description or null |
| `status` | `PENDING` or `COMPLETED` |
| `createdAt` | ISO-8601 timestamp |

## Relationships

No relationships. Users, ownership, authentication, and multi-tenant behavior are out of scope.
